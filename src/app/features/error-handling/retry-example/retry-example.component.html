What is retry?

The retry operator automatically resubscribes to the source Observable when an error occurs, attempting to complete successfully. It's useful for handling transient failures like network issues or temporary server problems.

When to Use

  • ✅ Handle temporary network failures
  • ✅ Retry failed HTTP requests automatically
  • ✅ Deal with intermittent server issues
  • ✅ Implement resilient data fetching

How It Works

1

Initial Attempt

The source Observable executes for the first time

2

Error Occurs

If an error happens, retry catches it

3

Resubscribe

Automatically resubscribes to the source Observable

4

Success or Exhaust

Either succeeds or exhausts all retry attempts

Real-World Use Cases

🌐 Network Requests

Retry failed HTTP calls automatically

http.get(url).pipe(retry(3))

📡 WebSocket Reconnection

Reconnect when connection drops

websocket$.pipe(retry({{ '{' }} delay: 1000 {{ '}' }}))

💾 Database Operations

Retry transient database errors

db.query().pipe(retry(2))

🔐 Authentication

Retry token refresh on failure

refreshToken().pipe(retry(1))

⚠️ Important Notes

  • Immediate retry: retry() resubscribes immediately without delay
  • Count parameter: Specifies maximum number of retry attempts
  • Use with delay: Consider using retryWhen for delayed retries
  • Idempotent operations: Best for operations that can be safely repeated

✅ Best Practices

  • Set reasonable retry limits to avoid infinite loops
  • Use retry for transient errors only
  • Combine with catchError for final error handling
  • Consider exponential backoff with retryWhen
  • Log retry attempts for debugging
