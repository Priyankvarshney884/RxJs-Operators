What is catchError?

The catchError operator catches errors on the Observable to be handled by returning a new Observable or throwing an error. It's essential for graceful error handling and recovery in reactive streams.

When to Use

  • ✅ Handle API errors gracefully
  • ✅ Provide fallback values when errors occur
  • ✅ Log errors and continue stream execution
  • ✅ Retry with alternative data sources

How It Works

1

Observable Executes

The source Observable starts emitting values normally

2

Error Occurs

An error is thrown in the Observable chain

3

catchError Intercepts

catchError catches the error and returns a fallback Observable

4

Stream Continues

The stream continues with the fallback data instead of terminating

Real-World Use Cases

🔄 Fallback Data

Return cached or default data when API fails

catchError(() => of(cachedData))

📝 Error Logging

Log errors and continue with empty result

catchError(err => {{ '{' }} logError(err); return of([]); {{ '}' }})

🔀 Alternative Source

Try backup API when primary fails

catchError(() => backupApi.getData())

🎯 User-Friendly Messages

Transform technical errors to user messages

catchError(err => throwError(() => 'Please try again'))

⚠️ Important Notes

  • Must return Observable: catchError must return an Observable (use of() or throwError())
  • Completes the stream: After catching, the original Observable completes
  • Position matters: Place catchError where you want to handle errors in the chain
  • Re-throwing errors: Use throwError() to propagate errors after logging

✅ Best Practices

  • Always provide meaningful fallback values
  • Log errors before handling them
  • Use with retry operators for automatic recovery
  • Consider user experience when choosing fallback strategy
  • Don't swallow errors silently - inform the user
