BehaviorSubject

Subjects Intermediate

A BehaviorSubject is a variant of Subject that requires an initial value and emits its current value to new subscribers. It always stores the latest value and provides synchronous access to it via the .value property.

  • Requires an initial value upon creation
  • New subscribers immediately receive the current value
  • Stores and provides synchronous access to the latest value
  • Perfect for representing state that always has a value

Key Characteristics

🎯 Initial Value

Must be created with an initial value that is immediately available to subscribers.

📡 Current Value

New subscribers immediately receive the most recent value upon subscription.

🔍 Synchronous Access

The current value can be accessed synchronously via the .value property.

💾 State Storage

Always maintains the latest emitted value, making it ideal for state management.

BehaviorSubject vs Subject

Feature Subject BehaviorSubject
Initial Value Not required Required
Late Subscribers Only receive new values Immediately receive current value
Synchronous Access No Yes (via .value property)
Use Case Event streams State management
Value Storage No Always stores latest value

Best Practices

✅ Do

  • Use for state that always has a value
  • Use for current user, settings, or configuration
  • Expose as Observable with asObservable()
  • Use for caching the latest value

❌ Don't

  • Use for event streams without meaningful initial value
  • Expose the BehaviorSubject directly (use Observable)
  • Use when you don't need the current value feature
  • Forget to complete when done
