AsyncSubject

Subjects Advanced

An AsyncSubject is a variant of Subject that only emits the last value of the Observable execution, and only when the execution completes. If the Observable errors, no value is emitted.

  • Only emits the last value upon completion
  • All subscribers receive the same last value
  • Late subscribers also receive the last value
  • Perfect for operations that produce a single final result

Key Characteristics

🎯 Last Value Only

Only the last emitted value is sent to subscribers, and only upon completion.

⏰ Waits for Completion

No values are emitted until the AsyncSubject completes.

🔄 Late Subscribers

Even late subscribers receive the last value after completion.

❌ Error Handling

If an error occurs, no value is emitted at all.

Best Practices

✅ Do

  • Use for operations that produce a single final result
  • Use for async operations like HTTP requests
  • Use when only the final outcome matters
  • Always call complete() to emit the value
  • Handle errors appropriately (no value on error)

❌ Don't

  • Use for continuous streams of data
  • Use when intermediate values are needed
  • Forget to complete (value won't be emitted)
  • Use for real-time updates or state management
  • Expect values before completion

AsyncSubject vs Promise

AsyncSubject

  • ✅ Can be cancelled (unsubscribe)
  • ✅ Multiple subscribers
  • ✅ Lazy (doesn't execute until subscribed)
  • ✅ Can be retried
  • ✅ Part of RxJS ecosystem

Promise

  • ❌ Cannot be cancelled
  • ❌ Single consumer (then)
  • ❌ Eager (executes immediately)
  • ❌ Cannot be retried natively
  • ✅ Native JavaScript
