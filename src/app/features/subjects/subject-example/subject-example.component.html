Subject

Subjects Intermediate

A Subject is a special type of Observable that allows values to be multicasted to many Observers. Subjects are like EventEmitters: they maintain a registry of many listeners.

  • Acts as both an Observable and an Observer
  • Can multicast values to multiple subscribers
  • New subscribers only receive values emitted after subscription
  • Does not have an initial value or replay behavior

Key Characteristics

🎯 Multicasting

A Subject can have multiple subscribers, and all of them receive the same values.

📡 Hot Observable

Subjects are "hot" - they emit values regardless of whether there are subscribers.

⏰ No Replay

New subscribers only receive values emitted after their subscription.

🔄 Bidirectional

Can act as both an Observable (subscribe) and an Observer (next, error, complete).

Subject vs Observable

Feature Observable Subject
Unicast/Multicast Unicast (each subscriber gets independent execution) Multicast (all subscribers share same execution)
Hot/Cold Cold (starts emitting on subscription) Hot (emits regardless of subscribers)
Can emit values No (only through operators) Yes (via next(), error(), complete())
Late subscribers Get all values from start Only get values after subscription

Related Subjects

BehaviorSubject

Subject with initial value and replay of current value

ReplaySubject

Subject that replays specified number of values to new subscribers

AsyncSubject

Subject that only emits the last value on completion
