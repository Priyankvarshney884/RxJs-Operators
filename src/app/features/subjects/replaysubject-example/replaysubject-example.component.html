ReplaySubject

Subjects Advanced

A ReplaySubject records multiple values from the Observable execution and replays them to new subscribers. You can specify how many values to replay and optionally a time window for the replay.

  • Replays a specified number of values to new subscribers
  • Can optionally specify a time window for replay
  • Useful for caching and replaying historical data
  • Perfect for scenarios where late subscribers need past values

Key Characteristics

📼 Value Recording

Records and stores a specified number of values from the Observable execution.

⏮️ Replay on Subscribe

New subscribers immediately receive all recorded values before receiving new ones.

🕐 Time Window

Optionally specify a time window - only values within the window are replayed.

💾 Buffer Management

Automatically manages buffer size, removing old values as new ones arrive.

Best Practices

✅ Do

  • Use for caching recent values or events
  • Set appropriate buffer size to avoid memory issues
  • Consider time window for time-sensitive data
  • Use for activity logs and history features
  • Complete the subject when done to free memory

❌ Don't

  • Use large buffer sizes without consideration
  • Use for infinite streams without time window
  • Forget about memory implications
  • Use when only current value is needed (use BehaviorSubject)
  • Expose the ReplaySubject directly (use Observable)
