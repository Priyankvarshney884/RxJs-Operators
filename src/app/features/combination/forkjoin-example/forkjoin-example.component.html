🔗 forkJoin Operator

🟡 Intermediate

What is forkJoin?

The forkJoin operator waits for all input Observables to complete, then emits an object/array with the last value from each. It's like Promise.all() for Observables - perfect for loading multiple resources in parallel.

When to Use

  • ✅ Load multiple independent resources in parallel
  • ✅ Dashboard initialization (fetch all data at once)
  • ✅ Parallel API calls that must all succeed
  • ✅ Combine results from multiple sources

🔍 Search Multiple Sources

Search across different APIs

📝 Form Submission

Save multiple related entities

forkJoin({{ '{' }} user, profile, settings {{ '}' }})

📥 Batch Downloads

Download multiple files in parallel

forkJoin(fileUrls.map(url => download(url)))

⚠️ Important Notes

  • All must complete: forkJoin only emits when ALL Observables complete
  • Error handling: If ANY Observable errors, forkJoin errors immediately
  • Last values only: Only the last emitted value from each Observable is used
  • Not for infinite streams: Don't use with Observables that never complete (like intervals)

✅ Best Practices

  • Use for parallel independent API calls
  • Add error handling with catchError for each Observable
  • Perfect for dashboard/page initialization
  • Combine with loading indicators for better UX
  • Consider timeout for slow requests
