What is zip?

The zip operator combines multiple Observables by pairing their emissions by index. It waits for each Observable to emit, then combines those values into an array or object. Like a zipper, it pairs values one-to-one.

When to Use

  • ✅ Match requests with responses
  • ✅ Pair related data streams
  • ✅ Synchronize parallel operations
  • ✅ Combine data by position/index

How It Works

1

Wait for All Sources

zip waits for each Observable to emit at least once

2

Pair by Index

Combines the 1st emission from each, then 2nd, then 3rd, etc.

3

Complete When Any Completes

Completes when the shortest Observable completes

Real-World Use Cases

📨 Request/Response Pairing

Pair related data from different sources

zip(userIds$, userDetails$)

🔄 Batch Processing

Process items with their metadata

zip(items$, metadata$)

⚠️ Important Notes

  • Slowest wins: zip is limited by the slowest Observable
  • Memory usage: Buffers values until all sources emit
  • Completion: Completes when ANY source completes
  • Strict pairing: Values are paired by index, not by time

✅ Best Practices

  • Use when order/pairing matters
  • Be aware of memory buffering
  • Consider combineLatest if you need latest values
  • Great for synchronizing streams
  • Completes when shortest Observable completes
