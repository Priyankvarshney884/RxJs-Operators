What is of?

The of operator creates an Observable that emits a sequence of values you provide as arguments, then completes. It's the simplest way to create an Observable from known values and is perfect for testing or providing static data.

When to Use

  • ✅ Create Observables from static values
  • ✅ Testing and mocking data
  • ✅ Provide default or fallback values
  • ✅ Emit multiple values synchronously

How It Works

1

Create Observable

of() creates an Observable with the provided values

2

Emit Values

Each value is emitted synchronously in sequence

3

Complete

After all values are emitted, the Observable completes

Real-World Use Cases

🧪 Testing

Mock data for unit tests

of(mockUser, mockProduct)

🔄 Default Values

Provide fallback data

catchError(() => of(defaultValue))

📋 Static Lists

Create dropdown options

of('Option 1', 'Option 2', 'Option 3')

🎯 Initial State

Set initial application state

of(initialState).pipe(tap(setState))

💡 Key Characteristics

  • Synchronous: All values are emitted immediately
  • Completes automatically: No need to manually complete
  • Multiple values: Can emit any number of values
  • Any type: Works with primitives, objects, arrays, etc.

✅ Best Practices

  • Use for known, static values
  • Perfect for testing and mocking
  • Combine with operators for complex flows
  • Use as fallback in error handling
  • Great for providing default configurations
