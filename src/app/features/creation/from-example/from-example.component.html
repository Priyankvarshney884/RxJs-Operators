What is from?

The from operator converts various data sources (arrays, promises, iterables, observables) into an Observable. It's incredibly versatile and essential for integrating different async patterns into the RxJS ecosystem.

When to Use

  • ✅ Convert arrays to Observables
  • ✅ Convert Promises to Observables
  • ✅ Work with iterables and async iterables
  • ✅ Integrate with other async patterns

How It Works

1

Identify Source

from() detects the type of input (array, promise, etc.)

2

Convert to Observable

Creates an Observable that emits values from the source

3

Emit Values

Each item is emitted sequentially

4

Complete

Observable completes after all values are emitted

Real-World Use Cases

🔄 Promise Integration

Convert fetch/async functions

from(fetch('/api/data'))

📋 Array Processing

Process arrays reactively

from([1, 2, 3]).pipe(map(x => x * 2))

🔀 Async Iteration

Work with async iterables

from(asyncGenerator())

🎯 Event Conversion

Convert event-based APIs

from(eventEmitter)

💡 from vs of

from()

Input: Array, Promise, Iterable

Behavior: Emits each item separately

from([1, 2, 3]) // emits: 1, 2, 3

of()

Input: Individual values

Behavior: Emits arguments as-is

of([1, 2, 3]) // emits: [1, 2, 3]

✅ Best Practices

  • Use from() for arrays and promises
  • Perfect for converting legacy async code
  • Combine with operators for powerful transformations
  • Handle promise rejections with catchError
  • Great for batch processing
