🟢 Beginner

What is debounceTime?

The debounceTime operator delays emissions from the source Observable until a specified time has passed without another emission. It's perfect for scenarios like search inputs where you want to wait for the user to stop typing.

When to Use

  • ✅ Search input (wait for user to stop typing)
  • ✅ Window resize events
  • ✅ Scroll events
  • ✅ Auto-save functionality

How It Works

1

User Types

Each keystroke emits a value from the form control

2

debounceTime Waits

The operator waits 300ms. If another keystroke occurs, the timer resets

3

Search Executes

After 300ms of silence, the search API call is made

Real-World Use Cases

🔍 Search Autocomplete

Wait for user to finish typing before searching

debounceTime(300)

💾 Auto-save

Save form data after user stops editing

debounceTime(1000)

📏 Window Resize

Handle resize events after user stops resizing

debounceTime(200)

📜 Infinite Scroll

Load more data after user stops scrolling

debounceTime(150)

✅ Best Practices

  • Use 300ms for search inputs (good balance)
  • Combine with distinctUntilChanged to avoid duplicate searches
  • Use switchMap to cancel previous requests
  • Adjust timing based on use case (faster for UI, slower for API)
  • Consider user experience - too long feels sluggish
