🔍 filter Operator

🟢 Beginner

What is filter?

The filter operator emits only those values from the source Observable that satisfy a specified predicate function. It's similar to Array.filter() in JavaScript.

When to Use

  • ✅ Filter data based on conditions
  • ✅ Remove null/undefined values
  • ✅ Conditional processing of events
  • ✅ Data validation and sanitization

How It Works

1

Source Observable

We start with an Observable that emits all user objects from the API

2

Apply filter Operator

The filter operator tests each user with a predicate function (age > 25 && active)

3

Filtered Output

Only users that pass the test are emitted to the subscriber

Real-World Use Cases

🛒 Product Filtering

Filter products based on price, availability, or category

filter(product => product.price > 0 && product.inStock)

🔐 Permission Checks

Only process events from authorized users

filter(user => user.role === 'admin')

🚫 Remove Null Values

Filter out null or undefined values from streams

filter(value => value != null)

📧 Email Validation

Only process valid email addresses

filter(email => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email))

✅ Best Practices

  • Keep predicate functions pure (no side effects)
  • Use filter early in the pipeline to reduce processing
  • Combine with other operators for complex filtering
  • Consider performance for large datasets
  • Handle edge cases (null, undefined, empty arrays)
