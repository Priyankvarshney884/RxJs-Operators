What is last?

The last operator emits only the last value (or the last value that meets a condition) emitted by the source Observable before it completes. It waits for the source to complete, then emits the final matching value.

Key Concepts

⏳ Waits for Completion

Must wait for source Observable to complete before emitting

🎯 Single Emission

Emits only one value (the last one) then completes

🔍 Optional Predicate

Can filter for last value matching a condition

⚠️ Error Handling

Throws error if no value matches (use default value to prevent)

Common Use Cases

  • Final Value: Get the last emission from a finite stream
  • Last Match: Find last item meeting criteria
  • Completion Result: Get final state after all operations
  • Validation: Check final value after processing

⚠️ Important Notes

  • Requires Completion: Source must complete, won't work with infinite streams
  • Memory: Keeps track of values until completion
  • Timing: Emission only happens after source completes
