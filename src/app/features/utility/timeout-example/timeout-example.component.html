What is timeout?

The timeout operator errors if the Observable does not emit a value within a specified time period. It's essential for handling slow or unresponsive operations, ensuring your application doesn't wait indefinitely for data.

Key Concepts

⏰ Time Limit

Sets maximum wait time for emissions

❌ Error on Timeout

Throws TimeoutError if time exceeded

🔄 Fallback Option

Can switch to fallback Observable instead of erroring

🎯 Per Emission

Can apply timeout to each emission or first only

Common Use Cases

  • API Requests: Prevent hanging on slow network requests
  • User Actions: Timeout waiting for user input
  • Real-time Data: Detect disconnected streams
  • Performance: Ensure operations complete within SLA

⚠️ Important Notes

  • Error Handling: Always use catchError to handle TimeoutError gracefully
  • Timing: Consider network latency and processing time when setting timeout
  • User Experience: Provide feedback when timeout occurs
  • Retry Strategy: Often combined with retry operators for resilience

💡 Timeout Strategies

First Emission

Timeout only for the first value

timeout(5000)

Each Emission

Timeout between each emission

timeout({{ '{' }} each: 5000 {{ '}' }})

With Fallback

Switch to fallback on timeout

timeout({{ '{' }} each: 5000, with: () => of('fallback') {{ '}' }})
