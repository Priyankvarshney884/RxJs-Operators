⏱️ delay Operator

🟢 Beginner

What is delay?

The delay operator delays the emission of items from the source Observable by a specified time span. All emissions are delayed by the same amount, maintaining the relative time intervals between them.

When to Use

  • ✅ Add artificial delays for testing or demos
  • ✅ Implement rate limiting or throttling
  • ✅ Create timed animations or transitions
  • ✅ Simulate network latency in development

How It Works

1

Source Emits

The source Observable emits a value

2

Delay Timer Starts

A timer is started for the specified delay duration

3

Wait Period

The value is held during the delay period

4

Delayed Emission

After the delay, the value is emitted to subscribers

Real-World Use Cases

🎬 Animations

Delay UI updates for smooth transitions

action$.pipe(delay(300))

🔄 Retry Logic

Add delays between retry attempts

retry$.pipe(delay(1000))

🧪 Testing

Simulate network latency in tests

mockApi$.pipe(delay(500))

⏰ Scheduled Actions

Delay actions to specific times

event$.pipe(delay(new Date('2024-01-01')))

💡 Delay Variants

⏱️ Time-based Delay

Delay by milliseconds

delay(1000) // 1 second

📅 Date-based Delay

Delay until specific date/time

delay(new Date('2024-12-31'))

🔀 delayWhen

Delay based on another Observable

delayWhen(() => timer(1000))

⚠️ Important Notes

  • Maintains order: Emissions maintain their relative timing
  • Delays all emissions: Every value is delayed by the same amount
  • Async operation: Uses setTimeout internally
  • Memory consideration: Long delays hold values in memory

✅ Best Practices

  • Use reasonable delay durations to avoid poor UX
  • Consider debounceTime for user input delays
  • Combine with retry for exponential backoff
  • Use for testing and development simulations
  • Be mindful of memory with long delays
