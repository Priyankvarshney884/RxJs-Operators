🔍 tap Operator

🟢 Beginner

What is tap?

The tap operator (formerly do) allows you to perform side effects for notifications from the source Observable without modifying the stream. It's perfect for debugging, logging, and triggering actions without changing the data flow.

When to Use

  • ✅ Debug and log Observable emissions
  • ✅ Update UI state (loading indicators, counters)
  • ✅ Trigger analytics or tracking events
  • ✅ Cache data without modifying the stream

How It Works

1

Value Arrives

A value is emitted from the source Observable

2

Side Effect Executes

tap performs its side effect (log, update state, etc.)

3

Value Passes Through

The original value continues unchanged to the next operator

4

Stream Continues

The Observable chain continues with the unmodified data

Real-World Use Cases

🐛 Debugging

Log values at different pipeline stages

tap(value => console.log('Debug:', value))

📊 Analytics

Track user interactions and events

tap(event => analytics.track(event))

💾 Caching

Store data in cache without modifying stream

tap(data => cache.set('key', data))

🔄 Loading States

Update loading indicators

tap(() => this.loading.set(false))

⚠️ Common Mistakes

❌ Don't modify the emitted value

// BAD - modifying the value
tap(user => {{ '{' }}
  user.name = 'Modified'; // Don't do this!
  return user;
{{ '}' }})
// GOOD - use map to transform
map(user => ({{ '{' }} ...user, name: 'Modified' {{ '}' }}))

❌ Don't use tap for transformations

// BAD - trying to transform in tap
tap(value => value * 2) // This doesn't work!

// GOOD - use map for transformations
map(value => value * 2)

✅ Best Practices

  • Use tap for side effects only, not transformations
  • Keep tap operations simple and fast
  • Don't modify the emitted values in tap
  • Perfect for debugging during development
  • Use for updating component state (loading, counters)
  • Avoid heavy computations in tap
