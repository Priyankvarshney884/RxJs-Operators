🏁 finalize Operator

🟢 Beginner

What is finalize?

The finalize operator executes a callback function when the Observable completes or errors, regardless of how it terminates. It's perfect for cleanup operations like hiding loading indicators, closing connections, or releasing resources.

When to Use

  • ✅ Hide loading indicators after requests complete
  • ✅ Clean up resources (close connections, timers)
  • ✅ Log completion or error events
  • ✅ Reset component state after operations

How It Works

1

Observable Executes

The source Observable starts emitting values

2

Values Flow Through

Values pass through finalize without modification

3

Completion or Error

Observable completes successfully or errors out

4

Cleanup Executes

finalize callback runs regardless of success or error

Real-World Use Cases

⏳ Loading States

Always hide loading indicator

finalize(() => this.loading.set(false))

🔌 Close Connections

Clean up WebSocket or HTTP connections

finalize(() => connection.close())

📊 Analytics

Track operation completion

finalize(() => analytics.track('operation_end'))

🧹 Resource Cleanup

Release resources and clear timers

finalize(() => clearInterval(timer))

💡 finalize vs Other Operators

🏁 finalize

When: On complete OR error

Use for: Cleanup operations

finalize(() => cleanup())

🔍 tap

When: On each emission

Use for: Side effects during stream

tap(value => log(value))

🛡️ catchError

When: Only on error

Use for: Error handling and recovery

catchError(err => of([]))

⚠️ Important Notes

  • Always executes: Runs on both complete and error
  • No return value: Cannot modify or return values
  • Unsubscribe safe: Also runs on unsubscribe
  • Order matters: Place at the end of your pipe for final cleanup

✅ Best Practices

  • Use for cleanup that must always happen
  • Perfect for hiding loading indicators
  • Place at the end of your operator chain
  • Keep finalize callbacks simple and fast
  • Don't throw errors in finalize callbacks
  • Use for logging completion events
